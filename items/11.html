11 Use final where applicable

  • final 은 field 가 재할당되지 않도록한다.
  • final 은 method 가 override 되지 않도록한다.
  • final 은 class 가 더이상 확장되지 않도록한다.
  • field 에서 final 을 쓰면 안되는 경우를 제외하고는 항상 final 을 붙여라
  • method 와 class 에서 final 을 써야만 하는 경우를 제외하고는 항상 final 을 붙여라
  • 가독성을 위해, parameter 와 local variable 에 final 을 붙이는 것은 선호하지 않는다. 그런데, Java 에서는 final 을 붙이는 것은 가독성을 떨어뜨릴 수 있지만, Kotlin 에서는 가독성을 떨어뜨리지 않는 듯하다.
