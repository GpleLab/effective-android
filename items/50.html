50 Use gradle wisely

Gradle is very powerful, use it for your every configuration and for dependencies of course.
Use manifestholder to have dynamic values in manifest 
manifestPlaceholders = [appName: APP_NAME]
Use resValue to add resources dynamically 
resValue "string", "RES_FOO", "RES FOO RELEASE"
Use applicationSuffix to change applicationId for different build types 
applicationIdSuffix '.qa'
Use buildConfigField to add more values dynamically in order to have different values for different build types/flavors 
buildConfigField 'String', 'ENDPOINT', '"YOUR_ENDPOINT"'
