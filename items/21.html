21 Use static factory method

  • You can have meaningful constructors. newIntent, newDialog, newAdapter etc
  • The class itself has the object creation responsibility
  • You can control the instance such as singleton
  • You can return subclass instead of the base type
  • You can keep the keys in the class and not expose
class LoginActivity {
    private static final KEY_FOO = "key_foo";

    public static Intent newIntent(Context context, String foo){
        Intent intent = new Intent(context, LoginActivity.class);
        intent.putExtra(KEY_FOO, foo);

        return intent;
    } 
}
And use it whenever you want to start LoginActivity 
class MainActivity {
    public void displayLogin(String foo){
        Intent intent = LoginActivity.newIntent(this,foo);
        startActivity(intent);
    } 
}
Same rule for adapters/fragments as well.
